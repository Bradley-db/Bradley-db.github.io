LeetCode At Hand (k-sum)

2019-03-11

0.问题回顾

回顾一下4sum问题的描述：

Given an array nums of n integers and an integer target, are there elements a, b, c, and d in nums such that a + b + c + d = target? Find all unique quadruplets in the array which gives the sum of target.

Note:

The solution set must not contain duplicate quadruplets.

Example:

Given array nums = [1, 0, -1, 0, -2, 2], and target = 0.

A solution set is:
[
  [-1,  0, 0, 1],
  [-2, -1, 1, 2],
  [-2,  0, 0, 2]
]

1.问题描述

给定一个包含 n 个整数的数组 nums 和一个目标值 target，判断 nums 中是否存在kkk个元素{n1,n2,n3,…nk}\{n_1,n_2,n_3,…n_k\}{n1​,n2​,n3​,…nk​} ，使得∑i=1kni\sum_{i=1}^{k} n_i∑i=1k​ni​ 的值与 target 相等？找出所有满足条件且不重复的kkk元组,其中k≥2k\ge2k≥2

注意：

答案中不可以包含重复的k​k​k​元组。

示例：

给定数组 nums = [1, 0, -1, 0, -2, 2]，和 target = 0，k = 4

满足要求的k元组集合为：
[
  [-1,  0, 0, 1],
  [-2, -1, 1, 2],
  [-2,  0, 0, 2]
]

2.解题思路

2.1 快速归类：

  • 数据结构：考察对数组知识的掌握程度；数组结构的相关问题主要考察如何高效遍历数组，需要重点关注任务的初始化条件和边界条件。
  • 任务类型：属于"遍历搜索类型”；

2.2 问题抽象重建：

  • 该题可以抽象理解为:提供一种算法，使得从给定整数集合中寻找任意k​k​k​个元素的非重复子集，该子集的元素的和与目标的值相等。
  • 问题的关键点在于如何快速寻找符合要求的子集.

2.3 思路：

  • 暴力搜索：复杂度O(nk)O(n^k)O(nk)
  • 启发式搜索+递归：
    • 当k=2k=2k=2时,可以参考2sum的算法在O(n)O(n)O(n)的时间复杂度内找到最最优解
    • 当k>2k>2k>2时,我们结合3sum,4sum问题的解题思路可以总结出一个规律：Ksum问题可以转换成外层的数组遍历和内层的求解(K-1)Sum问题。问题的时间复杂度O(nk−1)​O(n^{k-1})​O(nk−1)​

3.解法

3.1 排序+搜索

class Solution{
 private static List<List<Integer>> kSum (int[] nums, int start, int k, int target) {
        int len = nums.length;
        List<List<Integer>> res = new ArrayList<List<Integer>>();
        if(k == 2) { //two pointers from left and right
            int left = start, right = len - 1;
            while(left < right) {
                int sum = nums[left] + nums[right];
                if(sum == target) {
                    List<Integer> path = new ArrayList<Integer>();
                    path.add(nums[left]);
                    path.add(nums[right]);
                    res.add(path);
                    while(left < right && nums[left] == nums[left + 1]) left++;
                    while(left < right && nums[right] == nums[right - 1]) right--;
                    left++;
                    right--;
                } else if (sum < target) { //move left
                    left++;
                } else { //move right
                    right--;
                }
            }
        } else {
            for(int i = start; i < len - (k - 1); i++) {
                if(i > start && nums[i] == nums[i - 1]) continue;
                List<List<Integer>> temp = kSum(nums, i + 1, k - 1, target - nums[i]);
                for(List<Integer> t : temp) {
                    t.add(0, nums[i]);
                }
                res.addAll(temp);
            }
        }
        return res;
    }
}
